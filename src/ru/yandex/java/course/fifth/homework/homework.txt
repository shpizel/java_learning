1. Написать программу Ping-Pong. Один поток умеет говорить “Ping”, а другой “Pong”. Ни одно слово не должно повторять
два раза подряд. Пример:
Ping
Pong
Ping
Pong
…

2. Расширенная версия Ping-Pong. Но вместо двух состояний есть n(состояния обозначаются цифрой). За каждое состояние
отвечает отдельный поток. Поток умеет переводить состояние программы из предыдущего в новое.
К примеру, поток, который отвечает за состояние 5 может менять состояние программы из 4 в 5.
В результате на выходе программа должна пройти все состояния от 0 до n.
Входные параметры: n - количество состояний в системе, m - число циклов. Привет работы для n=3, m=2
0
1
2
0
1
2

3. Написать утилиту для тестирования функционала в многопоточной среде.
Классу можно указать какое действие нужно исполнять(Runnable), из скольки потоков и сколько раз его нужно повторить.
Утилита должна грамотно стартовать потоки(не должно быть такого, что первый поток создался и запустился,
а второй еще будет тратить время на создание) и быть синхронной.

+ в конце лекции есть краткое объяснение
