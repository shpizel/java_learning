1.1 Написать программу, которая будет переводить деньги между счетами пользователей. Необходимо использовать объектно-ориентированной подход.
Каждый класс должен отвечать только за одну функцию. Не должно быть такого, что логика по перечислению средств находится в том же классе, который описывает счет клиента.
1.2 Если перевести деньги со счета на счет не удалось - должен бросаться NotSufficientAmount exception.
1.3 Появляется два типа счетов. Один позволяет овердрафт, другой нет. Величина овердрафта должна задаваться при создании аккаунта.
2.1 Реализовать очередь FIFO (первым зашел - первым вышел) на основе массива String[], у которой есть два метода: положить объект в очередь и получить объект из очереди(при получении объект удаляется из очереди)
2.2 На основе предыдущего класса написать новый (воспользоваться наследованием), который хранит объекты в очереди заданное количество времени(время “жизни” объектов задается в конструкторе),
т.е. метод для получения объекта из очереди должен возвращать только те объекты, которые не превысил допустимый лимит “жизни”. Для отсчета времени можно воспользоваться классом LocalDateTime.
Получить текущий момент времени: LocalDateTime.now(). Пример вычитания времени: LocalDateTime.now().minus(1, ChronoUnit.HOURS), сравнения: LocalDateTime.now().plus(1, ChronoUnit.HOURS).isBefore(LocalDateTime.now())
